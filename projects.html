  • Mar15 - Jul15

    Sales Conversion Prediction

    - Android, Java, Python

    - At Novopay

    • Novopay is a business in the payments domain, that aims to achieve financial inclusion in India by empowering retailers(kirana shops) to service as banks by letting people do transactions using their Aadhaar identity.


    The Sales Conversion Prediction project was started with an aim to empower the sales team in making faster conversions by increasing their focus on promising retailers. This was achieved through collecting vast amounts of data about the retailer and his surroundings. The data was used to build a real-time conversion prediction model


    What did I worked on?

    • Data Collection & Feature Set Design: Data collection was a humongous task and needed an intelligent solution. I quickly learned Android and built a survey app that was embedded in the Novopay's retailer onboarding app. The survey collected 20 data points for each retailer. The survey was easily filled up in less than 2 minutes.

    • Exploratory Data Analysis & Preliminary Prediction Model: With the data that was collected, I conducted an exploration of the data to find out features that correlated with conversion. These features were a mixed group of categorical as well as continuous types. I used them to build a prediction model which was further taken up by the team to make realtime predictions.

    Tech Stack

    App: Android, Java, Protobuf, Mysql, Hibernate.

    EDA & Model: Python(scikit-learn and pandas), PMML.

  • Jun14 - Jul15

    Business/Ops Performance Dashboards

    - Elasticsearch, Kibana, Javascript, Java

    - At Khoslalabs & Novopay

    • Dashboards are an integral part of any operation that needs proactive realtime monitoring. During my stay at Novopay & Khosla Labs, I was responsible for the development of the analytics platform.


    Dashboards

    My aim at Novopay was to build dashboards that called for an action. Building such an experience was an iterative process. The hard-work put in culminated to dashboards that helped the business to:

    • React to failed transactions in real-time
    • View pending retailer applications by the minute and act on them
    • Monitor transaction/business growth with various metrics
    • View the activity of any sales-field-agent by the minute and optimize his/her workflow
    • Keep track of the technical infrastructure health

    These dashboards were built mostly on Kibana (along with a custom filtering proxy between Elasticsearch and Kibana to help with user role based filtering)


    Analytics Platform

    As part of the platform, I contributed to building a framework to allow for asynchronous pushing of events from various services to Elasticsearch. ES was chosen for its high availability, scalability and integration with Kibana. Data was pushed to ES in a fault tolerant way by using an in memory database (DerbyDB) in the event pipeline. For integration with custom dashboards, I contributed towards building a REST API based on the elasticsearch Java API

    Tech Stack

    Dashboards: Kibana, Javascript, D3.js

    Backend: Java, Elasticsearch, Apache Derby, mysql

  • Aug13 - Mar13

    Data Relay Server for Media Streaming on Mobile Devices

    - Python, AWS, C, TCP

    - At Mobitrix

    • Mobitrix was a startup founded in 2012 in the BYOD space with the aim of delivering users' desktop applications and content on their personal mobile devices.


    My main goal was to build scalable systems to support data streaming between multiple peers.

    Contributions:

    • Architected the communication model between the desktop streamer and the device from scratch
    • Implemented and integrated TCP hole punching to allow communication between peers located behind a NAT environment. Having a direct connection between peers helped reduce streaming load on our servers. But it was not always possible to achieve successful TCP hole punching.
    • Designed and implemented a data relay server to allow for cases where the routers did not allow TCP hole punching. The main challenges here were to write a custom load balancer on AWS and to modify the behaviour of the streamer from being a server to a client in real-time.

    Tech Stack

    Server: Python, C, TCP

    Infra: AWS
